# راه‌اندازیِ قابلیتِ موزیک

## ۱) چند تا اکانت واقعی درست کن (استخر یوزربات‌ها)
هرچقدر گروه بیشتری همزمان موزیک پخش کنن، به یوزربات بیشتری نیاز داری (یک
یوزربات فقط می‌تونه توی گروه‌هایی پخش کنه که خودش از قبل عضوشونه). برای هر
اکانت:

```bash
pip install telethon
python tools/generate_userbot_session.py
```

این اسکریپت **کاملاً مستقل** از ربات اصلیه - فقط لوکال اجرا می‌شه، شماره +
کدِ تایید + رمزِ دو-مرحله‌ای (اگه داشتی) رو می‌پرسه، و در آخر یک رشته‌ی
طولانی (StringSession) چاپ می‌کنه.

## ۲) متغیرهای محیطی رو در Render (یا `.env` لوکال) اضافه کن

```
USERBOT_API_ID=...          # از my.telegram.org - یک‌بار کافیه
USERBOT_API_HASH=...        # از my.telegram.org - یک‌بار کافیه
USERBOT_SESSIONS=session1,session2,session3   # هر رشته از مرحله‌ی ۱، با کاما جدا
```

می‌تونی با یک یوزربات شروع کنی و بعداً session جدید رو با کاما به همین
env اضافه کنی - نیازی به تغییرِ کد نیست.

## ۳) یوزربات‌ها رو به گروه‌ها اضافه کن
هر گروهی که می‌خوای توش موزیک پخش بشه، باید **حداقل یکی** از یوزربات‌های
بالا رو به‌عنوان عضوِ عادی (نه حتی ادمین) داشته باشه. اولین باری که کسی
توی اون گروه «پخش» بزنه، ربات خودش می‌گرده کدوم یوزربات عضوه و همیشه از
همون استفاده می‌کنه (نگاه کن به `music/pool.py`). اگه هیچ‌کدوم عضو نبودن،
ربات دقیقاً می‌گه کدوم یوزرنیم‌ها رو اضافه کنی.

دستور «یوزربات‌ها» (فقط برای مدیرانِ گروه) وضعیتِ استخر رو نشون می‌ده.

## ۴) دستورها (فقط owner/owner2/admin - بدون ویژه)
- **پخش** (ریپلای روی فایل صوتی/ویدیویی)
- **پخش آهنگ <اسم یا لینک یوتیوب>**
- **هاب** - پنل شیشه‌ای برای کنترل
- **مکث** / **ادامه پخش** / **بعدی** / **پایان پخش** / **شافل**
- **یوزربات‌ها** - وضعیت استخر

## ۵) نکته‌ی مهم درباره‌ی جستجوی یوتیوب (Render)
یوتیوب معمولاً آی‌پی سرورهای ابری (Render هم همین‌طور) رو به‌عنوانِ ربات
شناسایی و بلاک می‌کنه. برای کاهشِ این مشکل:
1. یک فایلِ `cookies.txt` (از افزونه‌ی مرورگرِ «Get cookies.txt» با یک
   اکانتِ واقعیِ یوتیوب) کنارِ ربات (کنارِ `bot.py`) بذار.
2. حتی با کوکی هم تضمینی نیست - یوتیوب می‌تونه IP رنجِ Render رو دوباره
   بلاک کنه. اگه این اتفاق افتاد، فقط حالتِ «ریپلای روی فایل» کار می‌کنه؛
   `music/youtube.py` در این حالت پیامِ خطای روشنی به کاربر می‌ده، نه سکوت.

## ۶) ffmpeg
پکیجِ `static_ffmpeg` (در requirements.txt) یک باینریِ ffmpeg رو موقعِ
استارتِ ربات خودکار دانلود می‌کنه - نیازی به تنظیمِ جداگانه‌ی buildpack
روی Render نیست (دقیقاً همون روشی که AnonAI-bot استفاده می‌کنه).
